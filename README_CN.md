# ai-app-driver

[English](README.md)

通过代码理解、能力提取、驱动层设计和任务技能，将现有应用转变为AI驱动的应用的实用方法。

## 这是什么

本项目探索了一条将现有应用转变为AI驱动应用的实用路径。

核心思想很简单：

AI不应仅仅与应用聊天、调用几个工具或点击UI元素。

相反，AI应该能够：

1. 阅读应用的代码库
2. 理解其结构和真实功能
3. 识别应用内部的稳定能力边界
4. 在这些能力基础上构建AI驱动层
5. 为实际工作生成任务特定的技能

这个存储库不仅是关于提示词、包装器或演示。  
它是尝试构建应用AI赋能的可复用方法论。

## 为什么存在this

如今，许多"为应用添加AI"的尝试在以下某个层级停止：

- 应用内的聊天UI
- 函数调用或工具
- UI自动化
- 没有稳定能力边界的智能体工作流

这些方法可能有效，但它们通常忽视了一个更重要的中层：

AI应该如何系统地理解应用、提取其可用能力，并将其转化为可维护的AI驱动层？

本存储库关注的正是这个缺失的层。

## 核心流程

当前的工作流程是：

1. **代码库理解**  
   AI读取并理解代码库结构。

2. **功能清单提取**  
   AI提取真实应用功能和可用能力。

3. **AI驱动层设计**  
   AI决定应用AI驱动层的结构方式。

4. **AI驱动层实现**  
   AI添加或生成驱动层代码。

5. **任务技能编写**  
   AI为真实工作编写面向任务的`skill.md`文件。

## 本存储库不是什么

本存储库不是：

- 特定模型的提示词集合
- UI自动化项目
- 简单的函数调用包装器
- 单个智能体演示
- 与某一语言、技术栈或供应商绑定的框架

## 当前范围

目前，本存储库专注于：

- 清晰地定义问题
- 提出可复用的工作流程
- 起草顶层工作技能
- 探索`skill.md`如何指导AI系统行为
- 收集示例、反例和实现案例
